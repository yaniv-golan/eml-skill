# `half` K=27 beam retry — null result

**Follow-up ID:** `P-half-k27-retry`
**Target:** shorten the shipped `half` witness from K=43 toward the paper's
direct-search K=27 claim (arXiv:2603.21852 Table 4, "half" row).
**Verdict:** no shorter tree found within our extended compute budget. The
paper's K=27 claim remains **unreproduced** by our tooling at feasible cost;
it is **not refuted** — K=27 is plausibly reachable with a sharper
canonical-form quotient or distributed enumeration that this retry didn't
attempt.

## Background

The shipped witness `half(x) = exp(sub(ln(x), ln(2)))` has K=43 and is an
upper bound. A prior beam attempt at
`max_k=27, per_level_cap=100k, seed-witnesses, 90s, right-half-plane`
enumerated 467k candidates without a hit. This retry **doubles the per-level
cap** (100k → 200k), **extends the time budget 20×** (90s → 1800s), and adds
**subtree seeding** on top of witness seeding so that intermediate structures
inside every non-target library witness get pre-populated at their own K
levels.

## Retry parameters

CLI:

```bash
python eml-skill/skills/eml-optimize/scripts/optimize.py search \
  --target half --domain right-half-plane \
  --per-level-cap 200000 --max-k 27 --time-budget 1800 \
  --seed-witnesses --seed-subtrees --strategy targeted --format json
```

Resolved beam knobs:

- `strategy = targeted` (meet-in-the-middle complement lookup enabled)
- `goal_depth = 2` (backward goal-set propagation; iter-4 default)
- `protected = True` (cap-eviction protects goal-set hits)
- `seed_witnesses = True` (iter-5: library witness trees seeded into `by_k`)
- `seed_subtrees = True` (iter-6: every internal subtree of every non-target
  witness seeded at its own K level)
- `retain_k = None` (no symbolic gate; `half` is not in `SYMBOLIC_TARGETS`
  so `sympy.simplify` fallback would not help)

## Interpretation

The per-level cap (200 000) saturates from K=17 onward. Every level from K=17
through K=27 enumerates a strict subset of its full search space — any K=27
`half` tree whose K=17..K=25 intermediate subtrees fall outside the retained
200k candidates at each level is invisible to this run.

Compared to the sqrt K=43 null-result (`docs/sqrt-k43-null.md`), `half`
shows the same cap-saturation pattern from K=17 on but *does not* exhibit
the goal-propagator's late-K collapse that sqrt saw at K≥31 — because the
max-k=27 ceiling arrives first. The beam simply runs out of K headroom.

## What would change the verdict

One or more of:

1. **Distributed enumeration.** Splitting the K=17..K=25 frontier across
   many nodes would let each shard retain 200k candidates without
   ever-growing single-machine memory. No code change required; it's an
   ops problem.
2. **A sharper canonical-form quotient.** The current numerical-equivalence
   hash (16-sample interior + branch probe) collides with false-negative
   rate that we pay for in cap pressure. A small-rewrite pre-hash (e.g.
   commuting `eml(a, 1) → exp(a)` leaves, folding `eml(1, 1) = e`) would
   shrink each K-level's unique-function count, freeing cap headroom.
3. **Backward-goal refinement tuned to `half`.** `goal_depth=2` is the
   iter-4 default that won on `mult` K=17. `half` may benefit from a
   deeper goal horizon, but that's an ablation worth its own run.

None of those are in scope for this retry.
