# Lower-bound framework sketch for EML primitive K values (2026-04-19)

## Motivation

The only minimality proofs currently shipped (`e` K=3, `exp` K=3, `ln` K=7, `zero` K=7, `sub` K=11, `minus_one` K=17, `mult` K=17, `two` K=19, `add` K=19) are obtained by **exhaustive enumeration** up to the witness K−2. This works because at small K the Catalan tree count is tractable: 626 arity-0 syntactic trees at K≤15, 28.1M arity-binary syntactic trees at K≤17 (`add`, ~10 min under the iter-7 memoized enumerator). But it does not scale:

| target   | library K | exhaustive enumeration at K−2                        |
|----------|----------:|------------------------------------------------------|
| `pow`    | 25        | ~60 s projected at K=23 binary (iter-7 scaling)      |
| `half_const` | 35    | ~910k trees attempted extended-reals, null           |
| `i`      | 75        | enumeration at K=73 infeasible — 2^36-ish trees      |
| `pi`     | 121       | enumeration at K=119 completely infeasible           |
| `sin`    | 351       | out of the question                                  |

For any primitive where shipped K > ~25 we cannot hope to upgrade `verdict="upper-bound"` → `"minimal"` by brute force. A **structural lower-bound theory** — an argument of the form "no K-tree of size ≤ N can represent target T because it violates invariant I" — would promote primitives cheaply.

This document sketches three candidate frameworks. Each section gives (a) the idea, (b) a toy worked example showing the framework is non-trivial, and (c) the concrete mathematical gap that stands between the sketch and a usable theorem.

---

## 1. Information-theoretic counting bound (Catalan × dedup)

### Idea

For arity-0 trees over leaf alphabet `{1}`, the number of syntactically distinct trees at K=2n+1 is the Catalan number `C_n`. Summed over all odd K ≤ K*, this is `Σ C_n` for n=0..(K*−1)/2. Every reachable constant value under IEEE principal-branch `cmath` is evaluated by at least one such tree; therefore

```
|{reachable constants at K ≤ K*}|   ≤   Σ_{n=0}^{(K*−1)/2} C_n
```

This bounds the cardinality of the reachable-value set, but on its own does not rule out any *specific* target. The sharper move is to attach a second invariant `φ: ℂ → S` (for some small set `S`) such that (i) every reachable EML value has `φ(v) ∈ R ⊆ S` (a structured subset), and (ii) `φ(target) ∉ R`. Then the target is unreachable at any K for which `R` still holds.

Equivalently: constrain the **image** rather than the preimage count. The counting side already tells us the image at K=15 is at most 626 values (observed: 355 distinct under `cmath`). The interesting theorems are those that restrict *which* 355 they can be.

### Toy example

Under `cmath`, `eml(a, b) = exp(a) − log(b)` takes a leaf alphabet `{1}` and produces only values in the sub-algebra **generated by `e` and `log`**:

- At K=1, reachable set = `{1}`.
- At K=3, reachable set = `{e − 0} = {e}`.
- At K=5, reachable set = `{e^e − 0, e^1 − log(e)} = {e^e, e − 1}`.
- At K=7, reachable = 5 values in `ℚ(e, e^e, log(e)) = ℚ(e, e^e, 1)`.

Every K=2n+1 value is a finite nested composition of `exp` and `log` applied to `1`. Call this the **elementary closure of `{1}` under `{exp, log}`**, denoted `E_1`. Schanuel's conjecture implies `e` is transcendental and `e^e` is algebraically independent from `e` — i.e., `E_1` is a transcendental tower. A target `−1` lies in `ℚ ⊂ E_1` *only* via one of the closure's algebraic-collapse events (e.g., `e − log(e) − log(e) + something = −1` by cancellation). These collapse events are rare and grow only at specific K levels where the tower acquires new degrees of freedom.

The `minus_one` K=17 result is one such collapse event. The natural conjecture is that the sequence of collapse-Ks for rational targets is sparse and computable — i.e., there is an explicit function `K_collapse(q)` for `q ∈ ℚ` such that `q` cannot appear as an arity-0 EML value below `K_collapse(q)`.

### Worked example: ruling out `i` at K ≤ N_real

Leaves = `{1}` (real), `exp` preserves reals-to-complex only via non-real arguments, and `log` of a positive real is real, `log` of a negative real is `log|·| + iπ`. **The first `π`-valued intermediate appears only once a tree nests `log` around a value whose real part has gone negative.** A candidate invariant:

```
φ(v) := (is-real(v), imag-class(v))
```

where `imag-class(v) ∈ {0, π·ℚ, other}`. Under K=3 the value `e` is real. Any *real* value entering `log` stays real (if positive) or produces `iπ + real` (if negative). Once `iπ` appears, `exp(iπ) = −1` is real again — so `i` itself requires constructing `iπ/2` or `log(−1)/2`, which demands a `½` constant, which itself has K=35 under current upper bounds and K=29 paper extended-reals.

A rigorous chain of the form

```
K(i) ≥ K(iπ) − 2 ≥ K(i · π) ≥ K(π) + K(i) − overlap
```

is self-referential, but the *structural* bound — that `i` requires a chain of `(log-of-negative)` events whose arguments themselves must be assembled — is the kind of recursion a theorem could close.

### Gap

The concrete mathematical gap is formulating `φ` so that it is **closed under `eml`**: if `φ(a) ∈ R` and `φ(b) ∈ R`, then `φ(exp(a) − log(b)) ∈ R`. Candidate invariants:

- **Algebraic degree over ℚ(e, π)**. Does `eml` bound the degree growth per level? (Plausibly: each node adds at most a fixed amount.) If yes, K → a lower bound on reachable algebraic degree, contrapositive gives a lower bound on K for high-degree targets.
- **Schanuel-style transcendence type**. More speculative; would need a specialization of Schanuel's conjecture that is provable for the specific generators `{1, exp, log}`.
- **A "log-depth" stratification**. The innermost `log` argument must itself be reachable; recurse. This might give a `K(log(v)) ≥ K(v) + Δ` style recurrence, similar to Kolmogorov-complexity recurrences.

If any of these can be made rigorous, it would promote `pi`, `i`, `half_const`, `sqrt` — all of which have `verdict="upper-bound"` — to either `"minimal"` (if the lower bound meets the shipped K) or at least to a sharpened `"upper-bound with proven K ≥ N"`.

## 2. Invariant-based bounds (algebraic-closure monitors)

### Idea

Find an algebraic invariant I : (subset of ℂ) → {true, false} such that:

1. I holds on every leaf `1`, `x`, `y`.
2. If I holds on `a` and `b`, then I holds on `eml(a, b)` — i.e., I is **closed under the EML operator**.
3. I does *not* hold on the target T.

Then no tree (of any K) with the shared leaf alphabet can reach T. To get a K-dependent bound, weaken to a **graded invariant** I_k where I_k closure holds when arguments sit at level k, and violation forces k-growth.

### Toy worked example: rationality shield for `half_const`

Let I(v) := "v ∈ ℝ and Im(v) = 0 under every evaluation branch". On leaves, I(1) holds. Under `eml(a,b) = exp(a) − log(b)`:

- If I(a) and I(b) hold and b > 0, then exp(a) is real positive and log(b) is real; I(eml(a,b)) holds.
- If b < 0, `log(b) = log|b| + iπ`; I(eml(a,b)) fails — but the violation is *specific*: the imaginary part is exactly `−π`.

Define `I_k(v) := v ∈ ℝ + π·ℚ·i` and track the **π-degree** of v (minimal k such that v ∈ ℝ + ℚ π i + ℚ π² + … with k total π-power factors). For any tree of K arity-0 tokens that stays in the π-free subring until the outermost nodes, the π-degree grows by at most 1 per `eml` node whose `b` argument was negative.

Targets like `½` have π-degree 0. If we could prove that `½` can only arise via a chain of cancellations `log(e^(1/2)) = ½` — which requires `e^(1/2)` as a sub-value, which itself is a specific transcendental that appears at some K_min — we have a K chain. This is almost exactly the argument paper Table 4 makes *implicitly* when publishing extended-reals K=29 for `½`: the construction is *via* `log(0)=−∞`, which IEEE rejects, and our lower bound is closed under IEEE semantics only.

### Gap

Two gaps:

1. **Invariant closure is usually too loose.** Rational-closure and algebraic-closure invariants don't exclude π or e because `exp` and `log` immediately transcend them. We need invariants that track *finer* structure — e.g., "lies in a specific transcendence-tower stratum" — and such invariants are typically conjectural (depend on Schanuel or weaker open questions in transcendence theory).

2. **Graded invariants must admit a monotone K-bound.** The claim `π-degree(eml(a,b)) ≤ max(π-deg(a), π-deg(b)) + c` for a fixed `c` would give a linear lower-bound recurrence. Whether such a `c` exists and is 0 on most operators (so that a π-degree-d target requires ≥ d `negative-b-log` events, each with their own K cost) is open. A first experimental step: enumerate all K ≤ 15 arity-0 trees, compute `Im(value) / π` for each, and check empirically whether it lies in a finitely generated ℚ-submodule. If yes, there's a concrete invariant to formalize.

### Experimental probe (cheap, unblocking)

Re-run the `minus_one` exhaustive enumerator but for every K ≤ 15 dump `(K, Im(v)/π, Re(v))` for each unique value. Check:

- Is `Im(v)/π` always in `ℚ` with bounded denominator?
- Does the bound on the denominator grow with K?

If the answer is "yes, bounded," we have an explicit graded invariant ready for proof.

## 3. Arity / composition lower bounds

### Idea

For a binary target like `pow(x, y) = x^y`, both arguments `x` and `y` must appear as leaves somewhere in the tree, and both must influence the output. The tree must therefore contain at least one sub-tree computing `log(x)` (any tree expressing `x^y = exp(y · log x)` must), at least one `exp`, and at least one `mult`-style combining node. Lower-bounds via inclusion:

```
K(pow) ≥ K(exp) + K(log) + K(mult-combiner) − overlap
       ≥ 3 + 7 + 17 − overlap
       = 27 − overlap
```

The library ships `pow` at K=25, and the paper publishes K=25 direct-search. The gap 27 − 25 = 2 is the *overlap budget*: how much can the three subtrees share nodes? Experimentally, the `pow` K=25 witness is

```
eml(eml(eml(1, eml(eml(eml(1, eml(eml(1, eml(1, y)), 1)), eml(1, eml(eml(1, x), 1))), 1)), 1), 1)
```

Unpacking: the `eml(1, eml(eml(1, x), 1))` subtree is `log(x)` at K=7. The `eml(eml(1, eml(eml(1, eml(1, y)), 1)), …)` scaffold implements `y * log(x)`. The outermost `eml(..., 1)` applies `exp`. Total node budget: 25, via **structural overlap** of 2 between the mult-scaffold and the log-scaffold (they share some constant `1` leaves).

### Toy example

A provable arity bound for a target that involves both x and y **non-trivially**:

- Every tree producing a binary function of x, y must have at least one sub-tree with an `x` leaf and at least one with a `y` leaf.
- At minimum, that sub-tree is a single leaf (K=1 each) combined by one operator (K=1 for the `eml` node).
- So K(any nontrivial binary) ≥ 2 + 1 = 3. (Hit exactly by `sub_naive = eml(x, y) = exp(x) − log(y)`, K=3 — but that's not the user's `x − y`; still, the *arity structure* is the floor.)

Strengthening: if the target requires `y` to be inside a `log` (for `pow`) or inside an `exp` (for `exp(y) + x`), then the path from the `y` leaf to the root must traverse at least one `log`-implementing sub-structure or one `exp`-implementing sub-structure. Counting the K cost of the cheapest such path gives an arity-aware lower bound.

### Gap

The obstruction is **node-sharing via aliasing**. Unlike a circuit where a gate output can be fanned out, an EML tree is a *tree* — every `log(x)` sub-expression needed in two places must be computed twice. Which means overlap is actually impossible: two independent occurrences of `log(x)` always cost 2×K(log). This makes the lower bound easier, not harder:

```
K(target) ≥ Σ (K(sub-expression) × multiplicity)
```

if each sub-expression's syntactic occurrence count can be argued. For `pow`, this gives `K(pow) ≥ K(log) + K(exp) + K(mult-stripped) = 7 + 3 + 9 = 19`, where `K(mult-stripped)=9` is the K of the mult scaffold minus its two leaf slots. 19 is a true lower bound — and it's loose by 6 versus the shipped 25. The interesting direction is: **which additional structural constraints tighten 19 → 25?** Conjectures:

- A `log(x)` sub-expression must have *x at a fixed depth* inside it; the two sub-expressions for `y · log(x)` cannot both rewrite the depth economy.
- The combining `mult` scaffold cannot share nodes with either the log or the exp sub-trees because the combining scaffold's two slots are consumed by their outputs.

If these are made rigorous (reducing to a graph-structural argument on the tree), K(pow) ≥ 25 follows and flips `verdict="upper-bound"` → `"minimal"` without enumeration.

This approach is most tractable for composite primitives (`pow`, `hypot`, `avg`, `log_x_y`) whose compiler forms are explicit.

## 4. Kolmogorov / description-length bounds (the most speculative)

### Idea

Treat a K-tree as a program of length K over a fixed 2-operator alphabet {leaf, eml-node} — effectively a binary encoding. The Kolmogorov complexity of any computable complex number T (to precision ε) is `H(T, ε) = log₂(1/ε) + O(1)`. For the tree encoding:

```
K_tree(T)  ≥  H(T, ε_cmath) / log₂(|alphabet|)
```

with `|alphabet| = 4` (three leaves `{1, x, y}` and the `eml` operator) — log₂ 4 = 2. And `ε_cmath ≈ 2^−50` (relative double precision floor is ~1e-15; as an additive precision on a unit-magnitude target, ~1e-15 ≈ 2^-50). So `H ≥ 50` bits, giving `K_tree ≥ 25` for any target requiring full double-precision distinguishability. That happens to match `pow`, coincidentally.

For a target that encodes *more information* than a double (e.g., a target "π to 10 extra bits that the tree must reproduce exactly under cmath" — which is not actually a well-posed spec under cmath float arithmetic), the bound would grow.

### Toy example

π to full cmath precision is `3.141592653589793`, 52 significand bits. `H(π, 2^-52) ≥ 52 − O(1)`. Under `log₂ 4 = 2`, `K(π) ≥ 26 − O(1)`. The shipped witness has K=121. The bound is **loose by roughly 5×** — useless.

### Gap

Kolmogorov-style bounds are fundamentally too weak for this application because the tree alphabet is small (4 symbols) and the targets are low-entropy (transcendentals with names). The useful form would require a **domain-specific description length** that recognizes EML structure: e.g., `H_EML(T)` where the code must respect eml-evaluation semantics. But `H_EML(T)` *is* K(T) — that's circular.

The only version of this bound that might bite is an **amortized form**: for a class of related targets (all constants, say), the *total* K across the class must exceed the total entropy. But targets are evaluated independently in practice.

This framework is the weakest of the three. Included for completeness; no non-trivial K promotion looks plausible via this route.

## Summary of promotion candidates

| framework                   | strongest plausible claim                                     | which witnesses it might flip |
|-----------------------------|----------------------------------------------------------------|--------------------------------|
| 1. Information-theoretic (transcendence-tower) | `K(algebraic target of degree d) ≥ f(d)` for explicit `f` | `minus_one` (re-prove), `two` (re-prove), `half_const` (promote) |
| 2. Invariant-based (π-degree / graded algebra) | `K(target with π-degree d) ≥ c · d + base`         | `i`, `pi`, `half_const`, possibly `sqrt` |
| 3. Arity / composition     | `K(composite P of {A, B, C}) ≥ K(A) + K(B) + K(C)` with explicit overlap budget | `pow`, `hypot`, `avg`, `log_x_y` |
| 4. Kolmogorov              | too loose to produce useful bounds                            | (none)                         |

If the π-denominators stay bounded by a low-degree polynomial in K, formalize that bound. Cost: ~1 hour of coding + analysis.

2. **Formalize overlap budget for framework 3 on `pow`.** Prove `K(pow) ≥ 25` via a tree-structural overlap argument specific to the log/exp/mult composition. Cost: probably 1–2 days of pen-and-paper + one referee check. High payoff: flips `pow` K=25 from `upper-bound` to `minimal` without needing K=23 binary enumeration.

3. **Survey transcendence literature for framework 1.** Specifically Lindemann–Weierstrass, Baker's theorem, and recent work on the arithmetic nature of iterated exp/log towers. There may be off-the-shelf lower bounds that apply to `E_1` (the elementary closure of {1} under {exp, log}). Cost: ~1 week, but may be obviated by (1) or (2).

4. **Skip framework 4.** Non-starter.

The minimum viable output from this sketch is (2): a proof that `pow` is minimal at K=25 via composition arity would be a genuine research deliverable and a practical reduction in the library's outstanding 🟡 entries.
